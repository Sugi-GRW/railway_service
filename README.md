# Railway Booking Service

Веб-сервис для поиска и бронирования железнодорожных билетов, разработанный на Django и PostgreSQL.

---

## Основные возможности

### 1. Умный поиск и фильтрация рейсов
* **Поиск в реальном времени**: Быстрый и удобный поиск рейсов по станциям отправления/прибытия и дате поездки.
* **Метки и сортировка**: Отображение специальных меток "Самый быстрый" и "Самый дешевый" для рейсов, а также возможность сортировки результатов поиска по времени отправления, времени в пути и стоимости.
* **Масштабное расписание**: Система оперирует базой данных, содержащей более **80 000 записей** о реальных маршрутах, промежуточных станциях, времени остановок и путях прибытия.

### 2. Интерактивная схема вагонов и выбор мест
* **Визуализация схем**: Динамическая прорисовка расположения мест для всех классов вагонов: **Сидячий, Плацкарт, Купе, СВ, Первый класс**.
* **Типизация мест**: Разделение мест на сидячие, нижние, верхние и боковые (в зависимости от типа вагона) с отображением стоимости с учетом индивидуального коэффициента комфортности вагона.

### 3. Бронирование и имитация оплаты
* **Условия оформления**: Для оформления билета пользователю необходимо войти в аккаунт и создать хотя бы один документ в личном кабинете.
* **Разные типы документов**: Поддержка оформления билетов на различные документы (Паспорт РФ, Свидетельство о рождении, Иностранный документ).
* **Имитация оплаты**: Встроенная система валидации банковских карт и имитации платежей, обеспечивающая переход билетов в статус подтвержденных.

### 4. Управление доп. услугами
* **Разнообразие сервисов**: Возможность заказа питания (нескольких видов), комплектов постельного белья, а также доступа в бизнес-зал.
* **Покупка доп. услуг после оформления билета**: Пассажиры могут докупить необходимые услуги к уже приобретенным билетам прямо из личного кабинета в любое время до отправления.
* **Защита от повторной покупки**: Система предотвращает повторную покупку услуг, которые могут быть приобретены только один раз (например, белья или доступа в бизнес-зал).

### 5. Подписка на уведомления
* **Подписка**: В случае отсутствия билетов пользователь может подписаться на уведомления об освобождении мест конкретного вагона на маршруте.
* **Email-уведомление**: При возврате билета другим пассажиром система автоматически проверяет список подписчиков, отсекает их по гендерному соответствию купе и отправляет им HTML-письмо на почту с прямой ссылкой на вагон для последующего выкупа освободившегося места.

### 6. Автоматический апгрейд
* **Автоапгрейд**: Уникальная отслеживаемая опция при покупке билета.
* **Фоновый планировщик**: Интегрированный в middleware фоновый процесс за 1 час до отправления поезда автоматически проверяет подтвержденные билеты с купленным автоапгрейдом.
* **Цепочка классов**: При наличии свободных мест в вагонах более высокого класса система автоматически делает апгрейд места!
* **Информирование**: Пользователь мгновенно получает HTML-письмо с информацией о новом вагоне и месте, а освободившееся место предлагается подписчикам этого вагона.
* **Безопасность**: Запуск апгрейда автоматически блокируется, если до отправления поезда осталось менее 70 минут.

### 7. Личный кабинет пассажира
* **Профиль пользователя**: Редактирование личных данных и контактной информации.
* **Шаблоны документов**: Удобная форма документов (пассажиров) для мгновенного заполнения данных при будущих покупках.
* **История поездок**: Управление активными и архивными билетами, просмотр QR-кода билета, а также детального списка подключенных дополнительных услуг.
* **Уведомления**: Отображение и управление своими активными подписками на уведомления о свободных местах.
* **Возврат билетов**: Процедура отмены билета с автоматическим запуском цепочки рассылок уведомлений для других желающих.
* **Удаление аккаунта**: Реализовано неполное (мягкое) удаление аккаунта — учетная запись деактивируется, но купленные билеты и документы пассажиров сохраняются в базе данных для отчетности.

### 8. Автоматическое завершение поездок
* **Фоновое отслеживание**: Встроенный планировщик автоматически переводит статус завершенных поездок в архивный статус «Выполнен» на основе точного времени прибытия поезда на конечную станцию маршрута.

### 9. Административная панель
* **Встроенная панель Django**: Использование стандартной панели администратора Django для удобного и полного управления пользователями, документами, билетами, станциями, рейсами и услугами.
* **Быстрое добавление рейсов**: Реализована автоматизированная процедура для быстрого и простого добавления новых рейсов в базу данных через встроенные скрипты.

## Стек технологий

* **Backend**: Python 3.10+, Django 6.x
* **Database**: PostgreSQL.
* **Frontend**: HTML5, Vanilla JavaScript (ES6+), Vanilla CSS3.
* **Mail System**: Gmail SMTP-интеграция для рассылки интерактивных адаптивных писем.

## Установка и запуск

Проект содержит автоматизированные скрипты развертывания БД и быстрого запуска сервера.

### Предварительные требования
1. Установлен **Python 3.10+**
2. Запущен сервер **PostgreSQL**
3. Создана пустая база данных с именем `railway_db`

---

### Шаг 1. Клонирование и настройка окружения

1. **Клонируйте репозиторий и перейдите в папку проекта**:
   ```bash
   git clone https://github.com/Sugi-GRW/railway_service
   cd railway_project
   ```

2. **Создайте и активируйте виртуальное окружение**:
   * **На Windows**:
     ```bash
     python -m venv venv
     venv\Scripts\activate
     ```
   * **На macOS / Linux**:
     ```bash
     python3 -m venv venv
     source venv/bin/activate
     ```

3. **Установите необходимые библиотеки**:
   ```bash
   pip install django psycopg2-binary
   ```

---

### Шаг 2. Настройка конфигурационного файла `.env`

Создайте файл `.env` в корневой папке проекта (рядом с `manage.py`) и укажите в нем пароль от вашей СУБД PostgreSQL.
Пример содержимого файла `.env`:
```env
DB_PASSWORD=qwerty123456
```
*(Параметры подключения к базе данных, такие как хост, порт и имя пользователя `postgres`, преднастроены в `config/settings.py`)*

---

### Шаг 3. Быстрое развертывание базы данных

Вместо ручного применения миграций и последовательного запуска множества скриптов, выполните **одну команду**, которая создаст всю структуру таблиц, зальет 80 000 записей маршрутов, добавит схемы вагонов и тестовые рейсы:

```bash
python full_deploy.py
```

*Скрипт `full_deploy.py` автоматически:*
1. Создает таблицы и связи (`schema.sql`).
2. Загружает полную базу маршрутов (`parsed_data.sql`).
3. Импортирует схемы вагонов и типы мест (`insert_layouts.sql`).
4. Генерирует демонстрационные рейсы между Москвой и Санкт-Петербургом (`add_moscow_spb_trip.sql`).
5. Корректирует последовательности первичных ключей (`fix_sequences.py`).

---

### Шаг 4. Запуск веб-сервиса

Вы можете запустить проект стандартной командой Django:
```bash
python manage.py runserver
```

**Для пользователей Windows** подготовлен скрипт быстрого запуска в один клик:
*Запустите файл **`run_server.bat`**. Скрипт активирует виртуальное окружение и поднимет локальный сервер.

После запуска откройте в браузере: **`http://127.0.0.1:8000`**

## Структура директорий проекта

```files
railway_project/
├── booking/                  # Основное приложение бронирования билетов
│   ├── admin.py              # Настройка панели администратора Django
│   ├── apps.py               # Конфигурация приложения booking
│   ├── context_processors.py # Процессор контекста для передачи данных пользователя в HTML-шаблоны
│   ├── email_utils.py        # Логика сборки и отправки адаптивных писем (покупка, возврат, апгрейд)
│   ├── middleware.py         # Фоновые проверки поездок и автоматический апгрейд
│   ├── models.py             # Описание ORM-моделей базы данных PostgreSQL
│   ├── services.py           # Вспомогательные паттерны бизнес-логики
│   ├── urls.py               # Контроллеры маршрутизации приложения
│   └── views.py              # Представления (основная логика обработки запросов)
├── config/                   # Конфигурационные файлы Django-проекта
│   └── settings.py           # Настройки проекта (БД, SMTP, Middleware)
├── database_setup/           # SQL скрипты и фиксы базы данных
│   ├── SQL files/            # Схемы таблиц, маршруты, расписание, лейауты
│   └── fix_sequences.py      # Скрипт синхронизации автоинкремента ID
├── templates/                # Папка HTML шаблонов
│   ├── emails/               # Красивые HTML-макеты писем для отправки пользователям
│   └── *.html                # Страницы личного кабинета, поиска, чекаута и т.д.
├── static/                   # Статические файлы (CSS стили, JS скрипты)
├── full_deploy.py            # Скрипт полного развертывания БД «из коробки»
├── run_server.bat            # Исполняемый файл быстрого запуска для Windows
└── manage.py                 # Главный консольный скрипт управления Django
```
